# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code에게 프로젝트 맥락을 제공합니다.

## 프로젝트 개요

**삼국유사 시간여행 지도** — 58개월(약 만 4~5세) 아이를 위한 삼국시대 인터랙티브 학습 사이트.
가야·신라·백제·고구려의 건국부터 멸망까지를 시간 이동 바 + 지도 + 인물 카드 형태로 보여준다.

- 대상 파일: `samguk-yusa-timeline-v2.html` (단일 HTML 파일, 외부 빌드 과정 없음)
- 기술 스택: Vanilla JS + 인라인 CSS + 인라인 SVG. 프레임워크 없음.
- 폰트: Google Fonts (`Do Hyeon` — 제목/버튼, `Gowun Dodum` — 본문), CDN 링크로 로드.
- 브라우저에 파일을 그대로 열면 동작해야 함 (서버/빌드 불필요).

## 파일 구조

단일 파일 안에 3개 섹션이 순서대로 존재:

1. `<style>` — CSS 커스텀 프로퍼티(`:root`)로 색상 토�300큰 정의, 반응형 그리드(`.stage`)
2. `<body>` — 헤더, 시간 이동 바(`#timelineTrack`), 지도 SVG(`#map`), 카드 패널(`#cardList`)
3. `<script>` — `ERAS` 데이터 배열 + 렌더링 로직

## 디자인 방향 (변경 시 유지할 것)

- **팔레트는 "고구려 고분벽화" 컨셉에서 파생됨**: 진흙빛 붉은색(`--clay`), 청록(`--teal`), 겨자빛 금색(`--gold`), 남색(`--indigo`), 짙은 밤하늘 배경. 크림+테라코타 조합(AI 생성 디자인의 흔한 기본값)은 의도적으로 피함.
- 국가별 고정 색상: 고구려=indigo, 백제=teal, 신라=clay-soft(코랄), 가야=gold, 외국/주변국=stone(회보라).
- 지도 도형에는 `feTurbulence` + `feDisplacementMap` SVG 필터(`#rough`)를 적용해 손으로 그린 듯한 붓터치 느낌을 줌. 새 지형을 추가할 때도 이 필터를 유지할 것.
- 타이포그래피: 제목/버튼/왕 이름표 = Do Hyeon(굵고 동글동글), 본문 카드 텍스트 = Gowun Dodum.

## 생성형 지도 이미지 스타일 기준

- 새 시대별 지도 이미지를 다시 생성할 때는 `assets/era-maps/era-07.png`의 톤을 기준으로 맞춘다.
- `era-07.png`처럼 전체가 너무 어둡거나 진하지 않은 지형 톤, 적당한 채도, 비교적 작은 지도 라벨, 얇은 경계선 느낌을 유지한다.
- `era-08.png`와 `era-11.png`는 이후 생성 이미지의 스타일 기준으로 삼지 않는다. 두 이미지는 글씨가 크고 색/대비가 진해져 기존 이미지들과 톤이 다르게 보일 수 있다.
- 전투 화살표나 보조 표시는 필요한 시대에만 추가하되, 지도 전체 톤과 라벨 크기는 `era-07.png`에 맞춘다.

## 생성형 전투 삽화 참고 자산

- 전투/군대 삽화를 생성할 때는 루트의 국기 이미지 3개를 항상 시각 참고로 삼는다.
  - `고구려_국기.png`: 고구려군 깃발/문양 참고. 붉은 바탕과 검은 삼족오풍 원형 문양을 고구려 깃발에 반영한다.
  - `백제_국기.webp`: 백제군 깃발/문양 참고. 백제군은 노랑 계열 포인트를 함께 사용한다.
  - `신라_국기.svg`: 신라군 깃발/문양 참고. 신라군은 파랑 계열 포인트를 함께 사용한다.
- 국기 이미지는 그대로 복붙하기보다, 시대극 삽화 안의 깃발·방패·군기 문양으로 자연스럽게 변형해 사용한다.
- 삽화는 어린이용 역사책 톤을 유지한다. 전투 장면이어도 피, 잔인한 부상, 시신, 공포 분위기는 넣지 않는다.

## 데이터 모델 (`ERAS` 배열)

각 시점(era) 객체 구조:

```js
{
  id, yr,          // 표시용 연도 문자열
  short,           // 타임라인 버튼에 들어갈 짧은 라벨 (줄바꿈은 \n)
  title,           // 카드 패널 상단 제목
  map: {           // 이 시점에 활성화할 국가별 지도 상태
    goguryeo: { v:'normal'|'peak',           m:'founding'|'normal'|'fading'|'fallen' },
    baekje:   { v:'normal'|'peak',           m:'...' },
    silla:    { v:'normal'|'peak'|'unified', m:'...' },
    gaya:     { v:'normal',                  m:'...' },
    // 활성화하지 않을 국가는 키 자체를 생략 (숨김 처리됨)
  },
  foreign: ['buyeo','huyeon', ...],  // 이 시점에 보여줄 주변국 라벨 id 목록 (선택)
  cards: [
    { k:'silla'|'goguryeo'|'baekje'|'gaya'|'foreign', king, icon, text, foreign?:true }
  ]
}
```

- `v`(variant) = 어떤 영역 모양(SVG path)을 쓸지, `m`(mood) = 그 모양에 적용할 CSS 상태(투명도/스케일/흑백 처리).
- `foreign:true`가 붙은 카드는 "다른 나라 이야기"로 점선 테두리 + 배지가 표시됨.
- 지도에 표시되는 왕 이름표(`king-tag`)는 `cards` 중 `foreign`이 아닌 카드에서 첫 번째 매칭되는 `king` 값을 사용함 (`render()` 참고).

## 지도 영역(SVG path) 확장 규칙

국가별로 미리 그려둔 영역 variant가 있고, 시점마다 그중 하나만 보이도록 토글한다 (`setKingdomState()`).

| 국가 | variant | 의미 |
|---|---|---|
| 고구려 | `normal` | 평상시(만주+평양 인근까지) |
| 고구려 | `peak` | 광개토대왕/장수왕 전성기(한강 유역까지) |
| 백제 | `normal` | 웅진·사비 시기, 남부 중심 |
| 백제 | `peak` | 근초고왕 전성기(북쪽까지 확장) |
| 신라 | `normal` | 진흥왕 이전, 동남부 소국 |
| 신라 | `peak` | 진흥왕 이후(한강 유역+가야 흡수+동해안 북상) |
| 신라 | `unified` | 676년 삼국통일 이후, 한반도 남부 전체 |
| 가야 | `normal` | 유일한 형태 (소국 연맹이라 변화폭이 작아 단순화함) |

새 시점을 추가하거나 영역을 조정할 때:
1. 반드시 위 표의 `v`/`m` 조합만 사용할 것 (새 variant 추가 시 `VARIANTS` 객체와 `<path id="t-{국가}-{variant}">`를 함께 추가해야 함).
2. `mood`는 CSS 클래스(`mood-founding|normal|fading|fallen`)로 매핑됨 — 새로운 mood를 추가하면 CSS에도 대응 클래스를 정의해야 함.
3. 지도 좌표계는 `viewBox="0 0 480 640"`. 실제 한반도 지리와 정확히 일치하지 않는 **의도적으로 단순화된** 도형이므로, 완벽한 지리적 정확도보다 "아이가 알아볼 수 있는 한반도 실루엣 + 시대별 크기 변화 체감"을 우선한다.

## 역사적 사실 검증 원칙

이 프로젝트는 실존 역사 인물/연대를 다루므로, 연대를 추가하거나 수정할 때는 **반드시 웹 검색으로 재검증**한다. 과거 검증에서 아래 오류를 실제로 발견/수정한 이력이 있음 (동일 실수 반복 주의):

- 신라 미추왕 재위: 262~284 (286 아님)
- 을지문덕 살수대첩: 612년 (613년 아님)
- 연개소문 정변(642)과 안시성 전투(645)는 별개 사건 — 뭉뚱그리지 말 것

확정된 연대 (재검증 완료, 근거는 대화 로그 참고):
근초고왕 346~375, 고국원왕 전사 371, 아신왕 392~405, 광개토대왕 391~413,
금관가야 멸망 532 / 대가야 멸망(이사부) 562, 연개소문 정변 642, 안시성 전투 645,
백제 멸망 660, 고구려 멸망 668, 신라 삼국통일 676, 장보고 청해진 828~841(보너스 시점).

## 타깃 사용자 관련 주의사항

- 문장은 반드시 짧고 쉬운 구어체(~해요체)로. 한자어·전문 용어 지양.
- 잔인한 묘사(전사, 죽음 등)는 사실은 유지하되 순화된 어휘로 표현 ("목숨을 잃었어요" 등).
- 이모지는 장식이 아니라 카드의 시각적 인덱스 역할(왕관=임금, 화살=장수 등 일관성 유지).

## 알려진 한계 / 향후 개선 아이디어

- 지도는 스키매틱(schematic)한 단순화 도형이며 실제 축척이 아님. 화면에도 "실제 국경선과는 다를 수 있어요" 안내 문구가 있음 — 제거하지 말 것.
- 가야는 소국 연맹체 특성상 세력 변화를 단순화(단일 `normal` variant)해 표현함. 더 정교하게 만들려면 금관가야/대가야를 별도 도형으로 분리하는 방법을 고려할 수 있음.
- 왕 이름표(`king-tag`)는 국가당 1개만 표시되므로, 한 시점에 같은 국가의 카드가 여러 개면 마지막 카드의 왕 이름이 덮어써질 수 있음(`render()`의 `forEach` 순서 문제) — 여러 왕을 동시에 표시해야 하는 시점을 추가할 경우 이 로직을 배열 기반으로 바꿔야 함.
